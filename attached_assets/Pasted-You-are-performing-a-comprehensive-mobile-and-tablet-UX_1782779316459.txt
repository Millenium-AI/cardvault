## GLOBAL MOBILE RULES (apply everywhere)

### Viewport & Safe Areas
- Ensure `<meta name="viewport" content="width=device-width, initial-scale=1, viewport-fit=cover">` is set in index.html
- All bottom-fixed UI must use `padding-bottom: env(safe-area-inset-bottom)` (already partially done in AppShell — verify it's applied correctly and consistently)
- Top content must respect `env(safe-area-inset-top)` on iOS (status bar overlap)
- Test at 320px (iPhone SE), 390px (iPhone 14), 430px (iPhone 14 Pro Max), 768px (iPad Mini), 1024px (iPad Pro)

### Typography
- Minimum body font size: 14px (never 10px–12px for readable content — the current `text-[10px]` labels in the bottom nav are the floor for icon labels ONLY)
- Headings on mobile: use `text-xl` max for page titles, `text-base` for section headers — avoid `text-2xl` and above on `sm:` breakpoint
- Truncate long strings with `truncate` or `line-clamp-2` rather than letting them overflow or wrap awkwardly

### Touch Targets
- Every interactive element (buttons, nav items, table rows, filter chips, form inputs) must be minimum 44×44px touch target (use `min-h-[44px]` or `py-3` padding)
- Card grid items and inventory rows need tap-friendly hit areas — no tiny icon-only buttons without padding

### Spacing
- Use `px-4` (16px) as the minimum horizontal page padding on mobile, `px-6` on tablet
- Stack verticals tightly but not claustrophobically: `gap-3` or `gap-4` between sections
- Avoid `p-6 md:p-8` patterns that cascade too much padding on small screens

Evaluate whether all 5 nav items can fit in a single bottom bar at 20px icon size with `text-[10px]` labels — if they fit without crowding at 320px, put them all in the bottom bar and remove the "More" drawer pattern entirely to simplify UX

- The `pb-20` on `<main>` must be dynamically calculated if bottom nav height changes — use a CSS variable `--bottom-nav-height` or Tailwind `pb-[calc(env(safe-area-inset-bottom)+56px)]` to be precise

### Tablet (768px–1024px)
- On tablet (`md:`), the sidebar is visible and collapsible — this is good. But at 768px the sidebar at `w-56` eats significant screen. Add a `lg:` breakpoint where the sidebar auto-expands: `md:w-14 lg:w-56` with icons only at `md:` and icons+labels at `lg:`
- Ensure main content area on tablet never goes below 600px wide

## INVENTORY — client/src/pages/Inventory.tsx (LARGEST FILE — most critical)

This is the core page. Read every section carefully before touching anything.

### Filter Bar
- The filter/search bar must become a sticky top bar on mobile that collapses non-essential filters behind an expandable "Filters" button
- Required always-visible on mobile: search input (full width), view toggle (grid/list)
- Behind "Filters" button tap: game filter chips, condition filter, sort dropdown, price range
- Filters panel should slide in from top or appear as a bottom sheet, NOT a modal that covers the full screen
- Filter chips must be horizontally scrollable: `flex overflow-x-auto gap-2 pb-2 scrollbar-hide` with `shrink-0` on each chip

### Card Grid (Grid View)
- Mobile: `grid-cols-2 gap-3` (two columns, tight but usable)
- Tablet: `grid-cols-3 sm:grid-cols-4`
- Desktop: `grid-cols-4 lg:grid-cols-6`
- Card thumbnails: maintain aspect-ratio `aspect-[2.5/3.5]` (standard card ratio) — use `aspect-[2.5/3.5] object-cover`
- Card name: `text-xs` with `line-clamp-2`, price below in `text-sm font-bold text-primary`
- Condition badge should sit as an absolute overlay: `absolute top-1 right-1`

### Card List (List View)
- Each row: card image thumbnail (`w-12 h-16 rounded object-cover shrink-0`), card name + set in a middle flex-1 column, price + condition badge right-aligned
- Full row tap area for detail view — no tiny edit/delete icons inline; use a swipe-to-reveal pattern or a tap to open a bottom sheet with actions
- Avoid horizontal overflow on list rows — price and condition must stay on the same row but truncate card names

### Card Detail / Edit Modal
- Must be a full-screen bottom sheet on mobile (`fixed inset-x-0 bottom-0 rounded-t-2xl max-h-[90dvh] overflow-y-auto`) rather than a centered modal
- Bottom sheet should have a drag handle bar at the top (`w-12 h-1 bg-muted rounded-full mx-auto mt-3 mb-4`)
- Inputs inside must be `text-base` to prevent iOS auto-zoom (iOS zooms in if input font-size < 16px)
- Action buttons (Save, Delete, Cancel) should be full-width stacked at the bottom of the sheet, outside the scroll area, with safe-area padding

### Bulk Actions
- Bulk action bar (select all, delete, export) should appear as a fixed bottom bar above the nav bar when cards are selected — `fixed bottom-[56px] left-0 right-0 bg-background border-t flex items-center gap-2 px-4 py-2`
- Checkboxes on cards in bulk-select mode should be `w-5 h-5` minimum
